\section{Kiến trúc tổng thể hệ thống}
\subsection{Giới thiệu tổng quan}

Hệ thống lưu trữ dữ liệu phân tán tích hợp nhận diện hành động qua video camera an ninh được thiết kế gồm các thành phần: Người dùng (Client), Hệ thống nhận diện hành động (AI Server), Hệ thống lưu trữ phân tán (Big Data Server), Name Node(Master), Data Node (Slave) và cơ sở dữ liệu metadata.

Hệ thống hoạt động trên mạng không dây và có dây. Có thể triển khai trên các máy tính cá nhân hoặc các máy tính nhúng.

Các thành phần giao tiếp thông qua giao thức HTTP/REST API.

Toàn bộ backend sử dụng Node.js, metadata quản lý bằng MongoDB. Nhận diện hành động sử dụng mô hình AI triển khai bằng Python (CLIP + LSTM).
\subsection{Mô tả các thành phần}
\begin{itemize}
    \item \textbf{Client (Người dùng)}: Phần mềm phát hiện chuyển động của camera và web quản lý file lưu trữ.
    \item \textbf{AI Server (Hệ thống nhận diện hành động)}: Máy chủ chứa mô hình nhận diện đã được huấn luyện dùng để nhận diện hành động được client gửi lên.
    \item \textbf{Big Data Server (Hệ thống lưu trữ phân tán)}: Hệ thống lưu trữ các video sau khi được nhận diện được gửi lên từ server nhận diện.
    \begin{itemize}
        \item \textbf{Name Node}: Trung tâm điều phối, quản lý metadata (thông tin về file, vị trí lưu trữ, trạng thái node), phân phối tác vụ đọc/ghi dữ liệu cho các Data Node.
        \item \textbf{Data Node}: Các máy chủ lưu trữ dữ liệu thực tế (video, file), thực hiện đọc/ghi theo lệnh từ Name Node, gửi heartbeat báo trạng thái.
        \item \textbf{Cơ sở dữ liệu metadata}: Lưu trữ thông tin quản lý file, block, trạng thái node và metadata phục vụ việc truy vấn nhanh.
    \end{itemize}
\end{itemize}

Name Node sẽ kiểm tra các Data Node còn hoạt động, thực hiện lưu metadata và trả về thông tin các Data Node còn hoạt động. Sau đó dữ liệu sẽ được lưu trực tiếp lên Data Node đc Name Node chỉ định. \subsubsection{CLIP model}
CLIP (Contrastive Language–Image Pretraining) là một mô hình học sâu mã nguồn mở do OpenAI phát triển, có khả năng kết nối thông tin giữa ảnh và văn bản. Mô hình được thiết kế để xử lý các tác vụ như phân loại hình ảnh theo văn bản mô tả (zero-shot image classification), tìm kiếm ảnh theo văn bản (text-to-image retrieval), và hiểu nội dung hình ảnh theo ngữ cảnh. CLIP được xây dựng bằng Python, sử dụng các framework như PyTorch, với hai thành phần chính: Text Encoder và Image Encoder, hoạt động song song để mã hóa ảnh và văn bản vào cùng một không gian vector.
\begin{figure}[H]
    \centering
    \includegraphics[width=1\linewidth]{Figure/AnhXaHinhAnhVaVanBanClip.png}
    \caption{Sơ đồ hoạt động của mô hình CLIP}
\end{figure}
Mô hình CLIP có 2 thành phần chính là Text Encoder và Image Encoder. Phần Text Encoder sử dụng Transformer - một kiến trúc cách mạng trong NLP từ năm 2017, nhận đầu vào là văn bản và mã hoá thành vector đặc trưng trong không gian embedding.\\
\textbf{Text Encoder}
    \begin{itemize}
        \item Chuyển văn bản thành các đơn vị token để xử lý tuần tự.
        \item Mỗi token sẽ được ánh xạ sang một vecto embedding cố định chiều để biểu diễn từ dưới dạng vector số học có thể tính toán được.
        \item Thêm thông tin vị trí của token trong câu vào embedding.
        \item Truyền qua các lớp transformer để nắm bắt ngữ nghĩa tổng thể và quan hệ giữa các tự trong câu:
        \begin{itemize}
            \item Multi-head Self Attention: Mỗi token có thể tập trung vào các token khác trong câu để hiểu ngữ cảnh.
            \item Feed Forward Network: Mỗi token được truyền qua một mạng con để tính toán đặc trưng sâu hơn.
            \item Layer Normalization và Residual Connection: Giúp quá trình lan truyền gradient ổn định và nhanh hơn.
        \end{itemize}
        \item Tóm tắt ngữ nghĩa toàn bộ câu thành một vector duy nhất bằng cách lấy vector của token [EOS] làm embedding đại diện cho toàn câu.
        \item Chuẩn hóa Embedding.
    \end{itemize}
\textbf{Image Encoder}
\begin{itemize}
        \item Tiền xử lý resize ảnh về 224x224 và đổi ảnh về dạng tensor sau đó chuẩn hoá pixel về đúng phân phối.
        \item Chia ảnh thành các patch nhỏ (32x32) sẽ được 49 patch, mỗi patch được flatten thành vector 1D và chuyển qua lớp linear projection để ebedding vector cùng kích thước 768 giúp transformer hiểu và xử lý dữ liệu hình ảnh như dữ liệu vector.
        \item Vì transformer không xử lý dữ liệu dạng không tuần tự như CNN nên cần thêm vector positional encoding vào mỗi embedding patch vector để đảm bảo thông tin vị trí không gian.
        \item Thêm token đặc biệt [CLS] để đại diện cho toàn bộ ảnh.
        \item Truyền qua các lớp transformer:
        \begin{itemize}
            \item Multi-head Self Attention: Mỗi token có thể tập trung vào các token khác trong câu để hiểu ngữ cảnh.
            \item Feed Forward Network: Mỗi token được truyền qua một mạng con để tính toán đặc trưng sâu hơn.
            \item Layer Normalization và Residual Connection: Giúp quá trình lan truyền gradient ổn định và nhanh hơn.
        \end{itemize}
        \item Lấy output embedding
    \end{itemize}
\subsubsection{LSTM}
LSTM (Long Short-Term Memory) là một biến thể của mạng nơ-ron hồi tiếp (RNN), được thiết kế để xử lý và học các chuỗi dữ liệu có độ dài thay đổi, đồng thời giải quyết vấn đề vanishing gradient thường gặp trong RNN truyền thống. LSTM đặc biệt hiệu quả trong việc ghi nhớ các thông tin quan trọng trong thời gian dài, nhờ vào cơ chế bộ nhớ (memory cell) và các cổng điều khiển.
Cấu trúc của LSTM bao gồm ba loại cổng chính:
\begin{itemize}
    \item \textbf{Cổng quên (Forget Gate):} quyết định thông tin nào trong trạng thái trước cần được loại bỏ.
    \item \textbf{Cổng vào (Input Gate):} xác định thông tin mới nào nên được thêm vào bộ nhớ.
    \item \textbf{Cổng đầu ra (Output Gate):} quyết định phần nào của trạng thái bộ nhớ sẽ được xuất ra ngoài làm đầu ra tại bước hiện tại.
\end{itemize}

Trong dự án này sẽ có 2 lớp LSTM:
\begin{itemize}
    \item Lớp LSTM 128 units tiếp nhận chuỗi vector đặc trưng từ ViT, từng bước xử lý thông tin ở từng time step (tương ứng với từng frame), đồng thời duy trì và cập nhật trạng thái ẩn (hidden state) và trạng thái bộ nhớ (cell state) nhằm lưu giữ các thông tin quan trọng về sự biến đổi của hành động trong video. Mục đích học các mẫu chuyển động ngắn hạn giữa các frame liên tiếp.
    \item Lớp LSTM 64 units nhận output từ lớp trước và tiếp tục xử lý, cho phép mô hình học được các mẫu chuyển động phức tạp hơn cũng như mối liên hệ dài hạn giữa các frame để khái quát và liên kết các chuyển động thành động tác hoàn chỉnh có tính logic và liên tục theo thời gian.
\end{itemize}

\section{Kiến trúc hệ thống lưu trữ dữ liệu phân tán}
\subsection{Tổng quan hệ thống}
Hệ thống được xây dựng dựa trên kiến trúc Master - Slave, trong đó Name Node có vai trò là Master chịu trách nhiệm điều phối các việc được giao và các Data Node là các slave chịu trách nhiệm thực hiện tác tác vụ được Name Node giao. \\
\textbf{Vai trò của Name Node (Master): }
\begin{itemize}
    \item Name Node không lưu trữ thực tế mà chỉ lưu trữ và quản lý metadata bao gồm:
    \begin{itemize}
        \item Tên tệp.
        \item Vị trí các khối dữ liệu.
        \item Các thuộc tính của tệp.
    \end{itemize}
    \item Điều phối các hoạt động ở Data Node như đọc/ghi dữ liệu.
    \item Theo dõi trạng thái của các Data Node (Check heartbeat).
\end{itemize}
\textbf{Vai trò của Data Node (Slave): }
\begin{itemize}
    \item Data Node lưu trữ dữ liệu thực tế dưới dạng các khối dữ liệu được phối trên nhiều Data Node khác nhau
    \item Thực hiện các lệnh đọc/ghi dữ liệu và chỉ mục tệp vào ổ đĩa từ phía người dùng thông qua NameNode.
    \item Gửi báo cáo trạng thái (Heartbeats) và danh sách khối dữ liệu (Block Reports) đến NameNode định kỳ.
    \item Nhân bản các khối dữ liệu sang các Data Node khác
\end{itemize}

\subsubsection{Metadata}

\textbf{1. Khái niệm Metadata}  

Metadata, hay còn gọi là siêu dữ liệu, là thông tin mô tả dữ liệu. Nó cung cấp ngữ cảnh cho dữ liệu thô, giúp định nghĩa, cấu trúc và tổ chức dữ liệu để hỗ trợ việc truy cập, quản lý và sử dụng hiệu quả hơn. Metadata có thể hiểu là “dữ liệu về dữ liệu,” chứa các thuộc tính như tên, loại, định dạng, nguồn gốc, hoặc mô tả dữ liệu.

Ví dụ: Trong một hệ thống tệp phân tán, metadata có thể bao gồm:  
\begin{itemize}
    \item Tên tệp.
    \item Vị trí các khối dữ liệu của tệp.
    \item Kích thước tệp.
    \item Quyền truy cập và thời gian chỉnh sửa gần nhất.
\end{itemize}

\textbf{2. Vai trò của Metadata trong hệ thống lưu trữ dữ liệu}  

Metadata đóng vai trò quan trọng trong quản lý và tối ưu hóa dữ liệu. Một số vai trò chính bao gồm:  
\begin{itemize}
    \item \textit{Tổ chức dữ liệu:} Metadata giúp phân loại và cấu trúc dữ liệu một cách hợp lý, dễ dàng tìm kiếm và truy cập.
    \item \textit{Quản lý dữ liệu:} Metadata cung cấp thông tin giúp quản trị viên theo dõi các thuộc tính và trạng thái của dữ liệu, như quyền truy cập hoặc lịch sử chỉnh sửa.
    \item \textit{Tăng hiệu suất:} Bằng cách lưu trữ thông tin về vị trí dữ liệu, metadata giúp hệ thống truy cập nhanh hơn mà không cần phải đọc toàn bộ dữ liệu gốc.
    \item \textit{Đảm bảo tính toàn vẹn và nhất quán:} Metadata hỗ trợ theo dõi trạng thái của dữ liệu trong hệ thống phân tán, đảm bảo sự đồng bộ giữa các bản sao.
\end{itemize}

\textbf{3. Các loại Metadata chính}  

Metadata được phân loại theo các chức năng sau:  
\begin{itemize}
    \item \textit{Metadata mô tả (Descriptive Metadata):} Cung cấp thông tin mô tả về nội dung dữ liệu, chẳng hạn như tiêu đề, tác giả, hoặc từ khóa.
    \item \textit{Metadata cấu trúc (Structural Metadata):} Xác định cấu trúc và mối quan hệ giữa các thành phần dữ liệu, ví dụ như thứ tự của các chương trong tài liệu hoặc liên kết giữa các tệp dữ liệu.
    \item \textit{Metadata hành chính (Administrative Metadata):} Cung cấp thông tin về quyền truy cập, lịch sử sửa đổi, hoặc dữ liệu kỹ thuật như định dạng hoặc kích thước.
\end{itemize}

\textbf{4. Ứng dụng của Metadata trong hệ thống phân tán}  

Trong các hệ thống lưu trữ dữ liệu phân tán, như Hadoop Distributed File System (HDFS) hoặc các hệ thống cơ sở dữ liệu NoSQL:  
\begin{itemize}
    \item \textit{Quản lý vị trí dữ liệu:} Metadata được sử dụng để theo dõi vị trí lưu trữ của các khối dữ liệu trên các node khác nhau.
    \item \textit{Tối ưu hóa truy cập dữ liệu:} Metadata giúp định tuyến yêu cầu đọc/ghi tới các node thích hợp, giảm độ trễ và tăng hiệu suất.
    \item \textit{Đảm bảo tính sẵn sàng:} Khi một node bị lỗi, metadata cho phép hệ thống tự động chuyển hướng yêu cầu tới bản sao dữ liệu ở các node khác.
\end{itemize}

\textbf{5. Thách thức trong quản lý Metadata}  

Dù có vai trò quan trọng, việc quản lý metadata cũng đối mặt với một số thách thức:  
\begin{itemize}
    \item \textit{Khối lượng metadata lớn:} Trong các hệ thống phân tán lớn, lượng metadata cần quản lý có thể rất lớn, đòi hỏi các chiến lược lưu trữ và xử lý hiệu quả.
    \item \textit{Độ trễ trong cập nhật:} Khi dữ liệu thay đổi, metadata phải được cập nhật ngay lập tức để đảm bảo tính nhất quán.
    \item \textit{Bảo mật:} Metadata thường chứa thông tin nhạy cảm, cần được bảo vệ khỏi truy cập trái phép.
\end{itemize}

Metadata không chỉ là thành phần hỗ trợ mà còn đóng vai trò cốt lõi trong việc quản lý dữ liệu, đặc biệt trong các hệ thống lưu trữ phân tán. Hiểu và quản lý tốt metadata giúp tăng hiệu quả vận hành và đảm bảo tính sẵn sàng, nhất quán của hệ thống.
